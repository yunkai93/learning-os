# Prompt 协议：课程包复盘

用于周期性路线校正。

## 什么时候复盘

- 一个 module 完成后
- 多次 session 失败后
- 长时间暂停后
- 学习目标变化时
- 开始 capstone 前

## 输出契约

在以下目录创建 review 记录：

- `journal/tracks/<track-id>/reviews/`

只有在用户明确同意时，才修订课程包 source 文件。

## 复盘问题

- 当前路线是否仍然对齐目标？
- outcome 是否过大或过虚？
- 证据质量是否足够？
- Codex 协助是否掩盖了薄弱技能？
- 是否有需要补救的反复错误？
- 节奏或 module 顺序是否应该调整？
