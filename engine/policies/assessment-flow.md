# 验收流程

## 目标

验收要判断三件事：学习者现在会什么、哪里需要补救、是否允许推进。

## 必需输入

- 当前 session 记录。
- 课程包 rubric source。
- 学习者提交的证据或 workspace 中的产物。
- 独立程度。

## 结果

允许的结果：

- `pass`
- `partial`
- `fail`

允许的下一步：

- `advance`
- `remediate`
- `repeat_smaller`
- `review_later`
- `revise_track`

## 规则

- `pass` 必须有足够证据，并且独立程度与 rubric 要求匹配。
- `partial` 表示证明了一部分 outcome，但不足以干净推进。
- `fail` 表示证据不可用，或暴露出缺失前置能力。
- Codex 代写不能直接让技能进入 mastered，除非随后有独立的变体任务或解释任务。
- 如果学习者在协助下完成了产物，要把产物状态和技能掌握状态分开记录。

## 进度更新

进度按 outcome 更新，允许状态：

- `not_started`
- `introduced`
- `practicing`
- `demonstrated_with_help`
- `demonstrated_solo`
- `mastered_for_now`
- `needs_review`
