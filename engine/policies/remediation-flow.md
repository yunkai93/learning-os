# 补救流程

## 什么时候补救

出现以下情况时进入补救：

- outcome 验收失败
- 缺少前置能力
- 只能在强协助下推进
- 重复错误暴露概念缺口
- 学习者要求放慢

## 补救方式

优先使用最小有效干预：

1. 解释缺失概念。
2. 给一个极小的引导练习。
3. 要求学习者复述。
4. 给一个变体任务。
5. 把原任务拆成更小检查点。
6. 回到某个前置 outcome。

## 规则

- 不跳过必需 outcome。
- 不把补救扩展成无关的大课。
- 小的前置缺口不需要新建课程包。
- 如果补救会影响进度，必须记录为 session 或 sub-session。
