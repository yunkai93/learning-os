# 全栈学习系统路线 Roadmap

## 目标

从 10 年前端经验出发，补齐 TypeScript 和后端能力，最终独立完成一个以 Learning OS 为主题的真实全栈项目，并达到可以胜任初级全栈工程师的水平。

这条路线不追求“尽快看完知识点”，而是按 evidence 和 rubric 推进。每个阶段都必须有可运行、可演示或可解释的成果，避免只产生“感觉会了”。

## 学习策略

- 早期 Codex 优先提示和检查，不直接代写。
- 卡住太久时允许 Codex 进入平衡协助，但必须记录独立程度。
- 每个 session 都要留下证据：命令输出、代码 diff、复述、变体任务或调试记录。
- Codex 代写不能算技能掌握，必须追加独立变体任务。
- 后端学习先从本体开始，再进入 Nest、数据库、鉴权和部署。

## 阶段

### 1. TypeScript 与 Node 工程入口

目标：理解 TypeScript + Node 项目的最小结构，能写基础类型和函数，能读懂常见 TS 报错。

产物：最小 TS + Node 程序，若干类型练习。

### 2. Node 文件、路径、JSON 与 CLI

目标：建立 Node 运行时基础，用本地 JSON 文件做持久化。

产物：学习日志 CLI，支持新增、查看、标记完成。

### 3. HTTP 与原生 Node API

目标：理解请求、响应、路由、状态码、Body、错误处理。

产物：原生 Node 学习记录 API。

### 4. Nest API 基础结构

目标：掌握 module、controller、service、DTO、validation、统一错误边界。

产物：Learning OS API v1，至少 3 个资源模块 CRUD。

### 5. PostgreSQL 与 Prisma 数据建模

目标：掌握表、关系、migration、关联查询、分页筛选和基础统计。

产物：持久化的 Learning OS API。

### 6. 鉴权、权限与真实业务规则

目标：实现注册登录、密码哈希、JWT、受保护接口、当前用户和服务层业务规则。

产物：带鉴权和基础权限的 Learning OS API。

### 7. React 前端联调与产品界面

目标：把后端 API 接成可操作的学习系统产品，补齐 TS 前端类型和请求封装。

产物：Learning OS 全栈应用 v1。

### 8. 测试、调试、部署与工程表达

目标：补齐关键测试、系统化调试、部署、README 和项目讲述。

产物：可复现、可部署、可讲述的项目版本。

### 9. Capstone 强化与求职化训练

目标：基于主项目做变体功能、限时任务、排错、重构和模拟面试。

产物：Learning OS Capstone 发布版。

## 主要项目线

1. 学习日志 CLI
2. 原生 Node 学习记录 API
3. Learning OS 后端 API
4. Learning OS 全栈应用
5. Learning OS Capstone 发布版

## 通过标准

最终通过不是“项目能跑”这么简单，而是同时满足：

- 能独立解释项目结构和技术取舍。
- 能独立新增相邻功能。
- 能定位前端、网络、后端、数据库、配置层问题。
- 能完成关键测试和部署演练。
- 能在限时任务中稳定输出。
- 能以初级全栈工程师候选人的方式讲清项目。
